Continuous Deployment

Continuous Deployment

With continuous integration after every commit...
we get build products after every commit

Why not deploy a passing build immediately?

This means smaller and quicker batches
which reduce risk and overhead

But there's a bit more we'll need for assurance...

Production Line

If a test fails...

reject the commit
report back
halt new commits

We want to be both fast and reliable

If we're failing tests, we're going too fast to be reliable

Deployment Script

Pushes new builds into production

Obeys the production line

If tests are failing...
no deployment until resolved

Otherwise...
deploy the changes

Real-time Alerting

Even if tests pass, things may still go wrong

Quick and accessible feedback via
monitoring and user reporting

Define an escalation process

If the pager goes off, production line halts
