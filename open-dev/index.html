Pull Request Workflow

  1. Developer branches from release tag
  2. Developer commits changes
  3. Developer submits pull request
  4. CI runs test suite and marks as pass/fail
  5. Team performs a code review and marks as pass/fail
  6. Integration Manager merges to release branch
  7. New version is tagged (later) from release branch
